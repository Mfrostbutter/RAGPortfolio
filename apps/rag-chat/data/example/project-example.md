# Example project writeup

Use this template for each project you want the AI to be able to answer about. One file per project is cleanest.

## Problem

What was broken or missing before this existed. Include rough scale if you have it (users affected, hours lost, dollars on the line).

## Approach

The decision that mattered. Not the full implementation — the key architectural call and why you made it over the obvious alternative.

## Result

What changed after it shipped. Numbers if you have them. Qualitative if you don't.

## Stack

Tools and services, just enough for the AI to be specific. Example: "FastAPI, Qdrant, Claude Sonnet 4, deployed to Cloudflare Pages."

## What I'd change

Optional but high-signal. Shows you reflect on your own work.
